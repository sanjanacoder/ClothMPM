# Data Card — Cloth Trajectories

> **Status:** placeholder. Populated in M2 (W4) by `src/mpm_cloth.py` + dataset generation script.

## Overview

Synthetic trajectory clips generated by our reference MPM simulator (NVIDIA Warp) on a 64×64 cloth sheet (~4096 particles).

## Scenarios

| Scenario | Target clips | Duration | Description |
|---|---|---|---|
| `drape` | ~4000 | 1–2 s | Cloth falls onto primitive shapes (sphere, cube, plane). |
| `wind` | ~3000 | 1–2 s | Cloth pinned at corners, wind direction/magnitude varied. |
| `collision` | ~3000 | 1–2 s | Sphere sweeps through pinned cloth. |
| **Total** | **≥10 000** | | |

## Per-frame schema

For each frame we save:
- `x` — particle positions, `(N, 3)` float32
- `v` — particle velocities, `(N, 3)` float32
- `a` — particle accelerations from the MPM grid update, `(N, 3)` float32 (target label)
- `F` — deformation gradient, `(N, 3, 3)` float32
- `f_ext` — external forces (gravity, wind, contact), `(N, 3)` float32
- `contact_flag` — boolean per particle, `(N,)` bool

Plus per-clip metadata: scenario name, seed, `dt`, scenario parameters (wind direction, sphere velocity, etc.), Warp version, MPM config hash.

## Storage

- One `.npz` per clip under `data/cloth_trajectories/<scenario>/clip_<seed>_<idx>.npz`.
- Manifest at `data/cloth_trajectories/index.csv` listing every clip with metadata.
- Total budget: ≲ 50 GB at ~5 MB/clip (revisit after M2).

## Splits

| Split | Seeds | Use |
|---|---|---|
| Train | 0–7 | Model fitting |
| Val | 100–102 | Early stopping, ablations |
| **Eval (frozen)** | 200–204 | All headline numbers; never seen during training |

## Determinism

Each clip can be regenerated bit-exactly from `(scenario, seed, params, mpm_config_hash)`. Verified by replay test in `tests/test_determinism.py` (M2).
