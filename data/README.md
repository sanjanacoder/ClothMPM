# Data Card — Cloth Trajectories

Synthetic trajectory clips generated by the reference MPM simulator (`src/mpm_cloth.py`, Taichi MLS-MPM) on a 64×64 cloth sheet (~4096 particles).

## Scenarios

| Scenario | Target clips | Duration | Description |
|---|---|---|---|
| `drape` | ~4000 | 1–2 s | Cloth falls onto a sphere; height and sphere position vary. |
| `wind` | ~3000 | 1–2 s | Cloth pinned at top corners; wind direction and magnitude vary. |
| `collision` | ~3000 | 1–2 s | Sphere sweeps through pinned cloth. |
| **Total** | **≥10 000** | | |

## Per-frame schema

For each frame we save:
- `x` — particle positions, `(N, 3)` float32
- `v` — particle velocities, `(N, 3)` float32
- `a` — particle accelerations from the MPM grid update, `(N, 3)` float32
- `F` — deformation gradient, `(N, 3, 3)` float32
- `contact_flag` — boolean per particle, `(N,)` bool

Plus per-clip metadata: scenario name, seed, `dt_s`, scenario parameters (initial height, sphere position/radius, wind force), MPM config hash.

## Storage

- One `.npz` per clip under `data/cloth_trajectories/<scenario>/clip_<seed>_<idx>.npz`.
- Manifest at `data/cloth_trajectories/index.csv` listing every clip with metadata.
- Estimated storage: ~5 MB/clip, ≲ 50 GB for a full 10k-clip dataset.

## Splits

| Split | Seed range | Use |
|---|---|---|
| Train | 0–999,999 | Model fitting |
| Val | 1,000,000–1,999,999 | Early stopping, ablations |
| Eval | 2,000,000–2,999,999 | Held-out evaluation; never seen during training |

For the smoke run, seeds 0–5 are used (2 per scenario) for fast schema verification.

## Determinism

Each clip can be regenerated bit-exactly from `(scenario, seed, params, mpm_config_hash)`. Verified by `test_smoke_determinism` in `tests/test_dataset.py`.
