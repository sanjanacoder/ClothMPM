# Risk and Ethics Memo (M1)

## Data and licensing

- **Training data is fully synthetic.** Every clip is generated by our own MPM reference (NVIDIA Warp) from `configs/mpm.yaml`. There is no upstream dataset license to inherit.
- **Dependencies.** Warp (Apache 2.0), PyTorch (BSD-3), PyTorch Geometric (MIT). All compatible with an Apache 2.0 release of this repo.
- **Papers.** Citations only; no copying of figures or text. PDFs in `papers/` are for personal study and never redistributed via this repo.

## Project risks (research)

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Warp's MPM cloth path is bumpier than expected (no off-the-shelf cloth sample) | Medium | M2 slips by 1 week | Allotted W3 checkpoint to fall back to Taichi MLS-MPM; both have the same downstream interface in `src/mpm_cloth.py` |
| Neural solver fails to maintain rollout stability beyond 1 s | Medium | H1 not satisfied | Curriculum from teacher-forcing → short rollouts; energy regularizer ablation; GNN backbone as Plan B |
| Detector cannot separate easy from hard frames at AUROC ≥ 0.8 | Medium | H2 not satisfied | Add strain-rate and ΔF channels; promote D2 → D3; report negative result honestly |
| Hybrid speedup is not 3× because fallbacks are too frequent | Medium | H3 not satisfied | F2 (mass-spring) is cheap; threshold sweep finds the operating point; if no point passes, report tradeoff curve |
| Colab session drops mid-training | High | Lost work | Checkpoint every 15–30 min; resume scripts; small enough models to retrain quickly |
| GPU memory pressure on T4 (16 GB on Colab Pro, 12-15 GB on free) | Low | OOM on large batches | Mixed precision and small batches; profile early in M3 |
| Determinism breaks (Warp non-deterministic kernels under reduction) | Low | Hard-to-debug eval drift | Determinism test in M2 catches regressions before they go anywhere downstream |

## Ethics and responsible use

- This is a research project on cloth simulation. There are no human subjects, no personally identifiable data, and no foreseeable dual-use concerns.
- Every reported number is **relative to our own MPM reference on a fixed eval set**. We do not claim "real time on any cloth", "production ready", or "matches commercial cloth solvers".
- Failure cases are reported in the manuscript with concrete named clips. Negative results are part of the deliverable.
- The cloth here is a generic 64×64 sheet with synthetic material parameters — no real garment, fabric brand, or person's clothing is modelled.

## What we will publicly release

- Source code (Apache 2.0).
- Reproducibility scripts that regenerate every dataset, table, and figure from a fixed seed list.
- Trained checkpoints (small enough to host on GitHub Releases or HuggingFace Hub).
- Manuscript PDF.

## What we will not release

- Pre-rendered demo videos with any third-party assets.
- Anything that depends on private internal infrastructure.
