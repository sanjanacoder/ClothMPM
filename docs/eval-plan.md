# Evaluation Plan (M1)

> Companion to [scope.md](scope.md). Locks the metrics, the eval set, and the protocol so that every later number is measured the same way.

## 1. Frozen evaluation set

Source: synthetic MPM trajectory clips generated by `src/mpm_cloth.py` from `configs/mpm.yaml`.

| Split | Seeds | Scenarios | Clip count | Purpose |
|---|---|---|---|---|
| Train | 0–7 | drape, wind | ~7 000 | Train neural solver |
| Val | 100–102 | drape, wind, collision | ~3 000 | Hyper-parameter selection, detector calibration |
| **Eval (frozen)** | 200–204 | drape, wind, collision | **15 clips** | All headline numbers |

The eval set is **never** loaded by any training script. Detector thresholds chosen on val are then evaluated unchanged on eval.

## 2. Primary metrics (definitions match Roadmap Appendix B)

Let `x̂_p(t)` be the predicted position of particle `p` at time `t`, `x_p(t)` the full-MPM reference, `N` particle count, `T` evaluation frames.

| Metric | Definition | What it tells us |
|---|---|---|
| **Per-particle L2** | `L2(t) = sqrt((1/N) Σ_p ||x̂_p(t) − x_p(t)||²)` | Pointwise drift vs. reference |
| **Chamfer** | Symmetric mean of nearest-neighbor distances between pred and ref point clouds | Insensitive to particle re-ordering |
| **Energy drift** | `|E_pred(t) − E_ref(t)| / E_ref(t)` | Stability / conservation proxy |
| **Rollout horizon** | `max{t : L2(t') ≤ τ ∀ t' ≤ t}` | "How long can we trust the rollout?" |
| **Cosine similarity** | `<a_t, a_{t-1}> / (||a_t|| ||a_{t-1}||)`, max-pooled over particles | Detector input D1 |
| **Wall-clock per step** | mean ± std over 1000 warm steps on Colab T4 | Speed |
| **End-to-end FPS** | 1 / (mean wall-clock per step) | Real-time claim |
| **Fallback fraction** | `(# physics-fallback steps) / (# total steps)` | How often does the safety net fire? |

## 3. Detector metrics

Per-frame binary label: `y_t = 1[L2(t) > ε]` where **`ε = 0.10 m`** (locked W2; see [m1-paper-notes.md](m1-paper-notes.md) §6).

- AUROC on the val and eval splits.
- FPR @ 95% TPR.
- Precision / recall at the chosen operating threshold.

## 4. Reporting protocol

- Every results CSV carries: `git_sha`, `config_path`, `config_hash`, `seed`, `scenario`, `clip_id`, `frame_idx`, plus the metric columns. This is enforced by `src/eval.py`.
- All headline numbers are computed by exactly one script: `python -m src.eval --config configs/eval.yaml`.
- All figures are produced by exactly one notebook each:
  - Figure 1 → `notebooks/20-hybrid-analysis.ipynb`
  - Figure 2 → `notebooks/20-hybrid-analysis.ipynb`
  - Figure 3 → `notebooks/20-hybrid-analysis.ipynb`
  - Table 1 → `notebooks/10-neural-solver.ipynb`
  - Table 2 → `notebooks/20-hybrid-analysis.ipynb`

## 6. Determinism and reproducibility

- All random seeds (`numpy`, `torch`, `warp`, scenario sampler) are set from `configs/eval.yaml::run.seed`.
- Each clip in `data/cloth_trajectories/index.csv` carries the seed and parameter dict needed to regenerate it bit-exactly.
- A regression test (`tests/test_determinism.py`, M2) re-simulates one clip per scenario and checks per-particle L2 < 1e-6 m vs. the saved trajectory.

## 7. Success bands (locked at M1, never moved later)

| Hypothesis | Pass condition |
|---|---|
| H1 | Per-particle L2 ≤ **τ = 0.05 m** for ≥1 s on `drape` and light `wind` (held-out eval) |
| H2 | Detector AUROC ≥ 0.8 on eval split (label `y_t = 1[L2(t) > ε = 0.10 m]`) |
| H3 | Hybrid wall-clock ≤ (1/3)× MPM wall-clock per step **and** per-particle L2 ≤ 2× pure-neural on non-fallback frames |

These bands are intentionally educational — we are not chasing SOTA. Failure to hit a band is still a publishable negative result if accompanied by an honest analysis.
